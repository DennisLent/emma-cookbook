Docker production flow is now driven by:

```sh
./scripts/setup_docker_production.sh
```

The interactive script asks for:
- production env file location
- public app URL and allowed hosts
- PostgreSQL credentials
- Django superuser credentials
- whether to import an existing backup (`.sql` or JSON app backup)
- whether to seed recipe data
- whether to run Ollama in Docker and which model to pull
- which Vosk model directory or zip URL to use
- JWT vs Keycloak settings

What it does:
- writes the chosen production env file
- prepares a persisted Vosk model in `docker-data/vosk`
- builds the Docker images
- starts Postgres and Redis first
- optionally starts Ollama and pulls the selected model
- optionally imports an existing database backup
- runs migrations and creates or updates the Django superuser
- optionally seeds recipe data
- starts backend, worker, and frontend containers

Keycloak notes:
- realm/client creation is still manual
- the script only collects and writes the backend env values
